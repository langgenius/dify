## 二、代码拉取

### 2.1 首次拉取（失败）
```bash
git clone https://github.com/langgenius/dify.git /home/project/dify
```
**问题**: RPC 失败，curl 56 Recv failure，连接超时
```
error: RPC 失败。curl 56 Recv failure: 连接超时
error: fetch-pack: unexpected disconnect while reading sideband packet
fatal: 过早的文件结束符（EOF）
```

### 2.2 解决方案
使用浅克隆（shallow clone）减少数据传输：
```bash

```
**结果**: 成功，下载了 28.49 MiB 数据

### 3.3 遇到的问题及解决

#### 问题 1: Nginx 端口冲突
**错误**:
```
Error response from daemon: failed to set up container networking:
failed to bind host port 0.0.0.0:80/tcp: address already in use
```

**原因**: 宿主机上已有 nginx 进程占用 80 端口

**解决**:
```bash
# 查找占用端口的进程
lsof -i :80

# 终止进程（根据实际情况）
sudo kill -9 <PID>
```

#### 问题 2: Docker 网络问题
**现象**: nginx 容器无法解析 "api" 主机
```
[emerg] host not found in upstream "api" in /etc/nginx/conf.d/default.conf:8
```

**解决**: 重启整个 Docker Compose 项目
```bash
docker compose down
docker compose up -d
```

## 五、遇到的问题汇总

### 5.1 模型配置错误

#### 错误信息
```
1 validation error for RuleGeneratePayload
model_config.mode
Input should be 'completion' or 'chat' [type=enum, input_value='agent-chat', input_type=str]
```

**原因**:
- `agent-chat` 是 Dify 的**应用模式**（AppMode），不是 **LLM 模式**
- 后端 `RuleGeneratePayload` 的 `ModelConfig.mode` 只接受 `chat` 或 `completion`
- Agent 应用底层调用 LLM 时应使用 `chat` 模式

**说明**: 这是 Dify 代码的问题，在使用 Agent 模式的应用进行提示词生成时会触发。如果需要修复，需要在前端代码中将 `agent-chat` 映射为 `chat`。

### 5.2 模型连接错误

#### 错误信息
```
An error occurred during credentials validation:
HTTPConnectionPool(host='localhost', port=11436):
Max retries exceeded with url: /v1/chat/completions
Connection refused
```

**原因**: 尝试连接本地 Ollama 服务但连接被拒绝

**解决**: 确保 Ollama 服务正在运行
```bash
ollama serve
ollama pull <model-name>
```

### 5.3 模型类型不匹配

#### 错误信息
```
Credentials validation failed with status code 404
{"detail":"Not Found"}
```

**原因**:
- 使用了 `Qwen3-Embedding-8B`（Embedding 模型）作为 LLM 使用
- Embedding 模型不能用于对话生成

**说明**:
| 模型类型 | 用途 | 示例 |
|---------|------|------|
| LLM | 对话生成 | Qwen2.5-7B-Instruct |
| Embedding | 文本向量化 | Qwen3-Embedding-8B |

### 5.4 插件安装失败 (signal: killed)

#### 错误信息
```
failed to launch plugin: failed to install dependencies:
signal: killed
output: Downloading transformers (11.4MiB)
init process exited due to no activity for 120 seconds
failed to init environment
```

**原因**:
1. **OOM Killer**: 进程被 Linux 内存管理杀死
2. **120 秒超时**: Sandbox 内部有"无活动120秒"超时限制
3. **下载慢**: `transformers` 等大型包下载/编译耗时过长

**已采取措施**:
- Sandbox 内存限制: 4G → 8G
- 配置国内清华 PyPI 镜像
- 延长 PLUGIN_PYTHON_ENV_INIT_TIMEOUT 至 600 秒

**状态**: 部分解决，仍可能失败

### 5.5 Web 容器 "Invalid environment variables" 错误

#### 错误信息
```
❌ Invalid environment variables: [
  {
    code: 'invalid_value',
    values: [ 'SELF_HOSTED', 'CLOUD' ],
    path: [ 'NEXT_PUBLIC_EDITION' ],
    message: 'Invalid input'
  }
]
```

**原因**:
1. `.env` 文件中 `CONSOLE_API_URL`、`APP_API_URL` 等 URL 配置为空，导致前端环境变量验证失败
2. `EDITION` 设置为 `CLOUD`，但 `NEXT_PUBLIC_EDITION` 只接受 `SELF_HOSTED` 或 `CLOUD`

**解决**:
```bash
# 配置所有必需的 URL
CONSOLE_API_URL=http://172.16.12.101
CONSOLE_WEB_URL=http://172.16.12.101
APP_API_URL=http://172.16.12.101/api
APP_WEB_URL=http://172.16.12.101

# 使用正确的 EDITION 值
EDITION=SELF_HOSTED
```

### 5.6 URL 路径重复导致 404

#### 错误信息
浏览器访问: `http://172.16.12.101/console/api/console/api/account/profile`
（路径中 `/console/api` 重复）

**原因**:
`CONSOLE_API_URL` 设置为 `http://172.16.12.101/console/api`，但前端会自动再添加一次 `/console/api`

**解决**:
```bash
# CONSOLE_API_URL 不应包含 /console/api
CONSOLE_API_URL=http://172.16.12.101
# 前端会自动拼接为: http://172.16.12.101/console/api/...
```

### 5.7 通知接口 404 错误

#### 错误信息
```
/console/api/notification 返回 404
```

**原因**:
- `EDITION=CLOUD` 时，Dify 会调用外部 BILLING_API_URL
- 自托管环境无法访问 Dify 官方服务器

**说明**: 这是预期行为，不影响核心功能。可忽略或配置 `EDITION=SELF_HOSTED`

### 5.8 用户注册需要邮箱验证码

#### 问题
开启了 `ENABLE_EMAIL_CODE_LOGIN=true`，但未配置 SMTP 邮件服务

**解决**:
```bash
# 关闭邮箱验证码，仅使用密码登录
ENABLE_EMAIL_CODE_LOGIN=false
ENABLE_EMAIL_PASSWORD_LOGIN=true
```

### 5.9 命令行创建管理员账号

当页面注册不可用时，可使用命令行创建账号：
```bash
cd /home/project/dify/docker

  --email admin@example.com \
  --name "Main Workspace" \
  --language zh-CN

# 输出示例:
```

## 七、后续建议

1. **插件安装**: 如果插件安装仍然超时，建议：
   - 使用轻量级插件
   - 在网络较稳定时重试
   - 或手动预安装依赖

2. **模型配置**: 确保使用正确的模型类型：
   - LLM 模型用于对话
   - Embedding 模型用于知识库检索

3. **邮箱配置**: 如需发送邮件验证码，需要配置 SMTP：
   ```bash
   MAIL_TYPE=smtp
   MAIL_DEFAULT_SEND_FROM=your_email@qq.com
   SMTP_HOST=smtp.qq.com
   SMTP_PORT=587
   SMTP_USERNAME=your_email@qq.com
   SMTP_PASSWORD=your授权码
   SMTP_USE_TLS=true
   ```

4. **定期更新**: 如需更新 Dify 版本：
   ```bash
   cd /home/project/dify
   git pull
   cd docker
   docker compose down
   docker compose pull
   docker compose up -d
   ```
