# Frontend Human Input coupling inventory

This inventory records the frontend boundaries that currently key on `BlockEnum.HumanInput`, `HumanInputNodeType` or v1 delivery methods. Human Input v2 must be version-routed at editor boundaries and must not enter existing runtime-only v1 paths.

| Area                        | Current coupling                                                                                                                                                        | v2 treatment                                                                                                                                              |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Catalog and metadata        | `workflow/constants/node.ts`, `block-selector/*`, `hooks-store`, `use-nodes-meta-data.ts`, `use-available-blocks.ts`, `start-placeholder/panel.tsx`                     | Add an independent UI-only catalog entry; central catalog resolver maps exact v2 DSL to it; normalize created data back to `type: human-input`            |
| Node and panel maps         | `workflow/nodes/components.ts`, `_base/node.tsx`, `_base/components/workflow-panel/index.tsx`, `block-icon.tsx`                                                         | Route persisted Human Input through an exact-version wrapper; keep direct UI identity mapping for candidate rendering                                     |
| Validation and checklist    | `human-input/default.ts`, `hooks/use-checklist.ts`, metadata lookups                                                                                                    | Register an independent v2 validator and route through `getNodeCatalogType`; leave v1 delivery validation unchanged                                       |
| Shared editor UI            | `human-input/panel.tsx`, `components/form-content*`, `components/user-action.tsx`, `components/timeout.tsx`, `hooks/use-config.ts`, `hooks/use-form-content.ts`         | Extract a narrow shared shape/hook/sections for form content, inputs, actions and timeout; v2 never imports `delivery_methods`                            |
| Branch handles and layout   | `human-input/node.tsx`, `use-nodes-interactions.ts`, `utils/elk-layout.ts`, `utils/workflow.ts`                                                                         | Consume the shared action/timeout shape; treat the UI-only identity as branch-capable during creation while persisted edges continue to use `human-input` |
| Outputs                     | `human-input/default.ts`, `workflow/constants.ts`, output panels and metadata `getOutputVars`                                                                           | Share input-derived outputs; preserve `__action_id`, `__action_value`, `__rendered_content` and `__timeout` behavior                                      |
| Variables and dependencies  | `_base/components/variable/utils.ts` and tests                                                                                                                          | Add v2 dynamic recipient selectors and message-template tokens; apply rename/delete/remap without calling v1 delivery-method transforms                   |
| Copy, paste and node change | `hooks/use-nodes-interactions.ts`, clipboard helpers and tests                                                                                                          | Resolve defaults by version-aware catalog type; preserve discriminator, typo wire key, nested values and order                                            |
| Edge cleanup                | `human-input/hooks/use-config.ts`, `use-edges-interactions`                                                                                                             | Share action add/rename/delete behavior so v1/v2 update and remove the same source handles                                                                |
| Workflow availability       | `use-workflow.ts`, `next-step/*`, selection/context menus, snippet insertion                                                                                            | Include the UI-only identity wherever a newly selected candidate is evaluated; normalized persisted nodes remain Human Input                              |
| Single-run and runtime      | `_base/components/workflow-panel/last-run/use-last-run.ts`, `_base/hooks/use-one-step-run.ts`, `before-run-form`, `use-workflow-run-event`, workflow run log formatting | Keep v1 behavior intact. v2 must not call these Human Input runtime hooks and must not simulate execution                                                 |
| End-user response rendering | base chat Human Input content/form list, workflow panel Human Input form list, workflow preview nodes                                                                   | Existing server-result rendering remains v1/runtime-owned; no new v2 execution/result path is introduced                                                  |
| Tests                       | existing Human Input node/panel/hooks/components, workflow variables, ELK, interactions, preview and run-format tests                                                   | Preserve all existing suites and add exact guard, defaults, round-trip, shared behavior, recipient, debug, template and router coverage                   |

## Narrow shared shape

Only these fields are shared by v1 and v2 editor infrastructure: `form_content`, `inputs`, `user_actions`, `timeout`, and `timeout_unit`. `delivery_methods`, `recpients_spec`, `message_template`, `debug_mode`, and `version` stay version-specific.
