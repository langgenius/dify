三条执行管线： Fast = ordinary hybrid + final rerank；Research = Summary/Outline/PageIndex，Graph 与 ordinary rerank zero-call；Deep = ordinary hybrid + Graph expansion + one unified final rerank。
Top K / Threshold： Fast、Deep 在最终 rerank score 上先 threshold 再 Top K；Research 在 versioned PageIndex normalized score 上先 threshold 再 Top K。权限过滤与 immutable publication closure 对三条管线都生效。
Auto 不是第四条管线：/queries 接受显式 auto，由空间 published reasoning model 选择 concrete mode；未传 mode 使用空间 defaultMode。路由模型失败回落同一 default，不运行旧启发式。
可追踪性：在线查询将 requested/resolved/resolver/model/prompt/degraded 等 bounded provenance 写入 AnswerTrace 的 query.route step；异步 Research create 冻结并持久化一次 Auto 决策，worker retry/restart 不重判。
答案生成：三条管线最终都使用知识空间 frozen published reasoningModel；模型/provider/credential 不从部署级默认值猜测。
/fs/* 命令： ls/tree/find/stat/cat/open_node/grep 都是确定性路径/仓库读取,走 knowledge_paths 视图 + nodes/segments,不经 embedding/RRF/rerank/planner,也无 fast/deep/research。唯一推理 hook 是 fs/diff 的可选 semanticDiffProvider(LLM),apps/api 未接线。⚠ 这些视图依赖 knowledge_paths 已物化——当前 loader 未建(0 行)→ ls 视图为空。
